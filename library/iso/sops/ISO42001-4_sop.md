# SOP — AI Context — Organization and Interested Parties
**Process ID:** ISO42001-4
**Framework:** ISO 42001:2023 | **Domain:** ISO 42001
**Generated:** 2026-06-10

## Purpose
Understanding the organizational context for AI management including AI-specific interested parties, regulatory environment, AI use case scope and AIMS (AI Management System) boundary definition

## Triggers
- New AI use case added to inventory
- Regulatory update published
- Annual AIMS review scheduled

## Inputs Required
- AI use case inventory
- regulatory requirements
- stakeholder AI expectations
- organizational AI strategy

## Process Steps
1. IF RegulatoryRequirement contains EU AI Act THEN include high-risk classification in AIContextAnalysis
2. IF StakeholderAIExpectation includes external regulators THEN add regulator to AIInterestedPartyRegister
3. IF sector is defense THEN enforce additional security boundary in AIMSScope

## Expected Outputs
- AIMS scope
- AI interested party register
- AI context analysis
- AI use case register

## Business Rules
- All inputs must be validated before producing outputs
- AIUseCaseRegister must cover 100% of AIUseCaseInventory
- ComplianceFlag ISO 42001:2023 Clause 4 must be set true on completion
- automation_potential threshold 0.55 triggers semi-automated workflow

## Exception Handling
- Missing OrganizationalAIStrategy: halt process and request strategy document from executive sponsor
- Non-applicable sector: log exclusion and continue with remaining sectors

## Success Criteria
- AI use case coverage KPI >= 0.95
- stakeholder engagement rate >= 0.80
- All compliance_flags set to true
- AIMSScope and AIInterestedPartyRegister approved by process owner

## Compliance Requirements
- ISO 42001:2023 Clause 4
- EU AI Act context assessment
- GDPR AI context
