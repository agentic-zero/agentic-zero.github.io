# SOP — Develop Design Chain Roadmap
**Process ID:** SCOR-D1.2
**Framework:** SCOR-D | **Domain:** Design Chain
**Generated:** 2026-06-05

## Purpose
Process of creating a detailed roadmap for the design chain, including key milestones and deliverables

## Triggers
- Design Chain Strategy update
- Change in Resource Availability
- Project Schedule update

## Inputs Required
- design chain strategy
- design chain objectives
- resource availability

## Process Steps
1. IF Design Chain Strategy is updated THEN re-evaluate Design Chain Roadmap
2. IF Resource Availability changes THEN adjust Design Chain Roadmap
3. IF Milestone Achievement is behind schedule THEN re-prioritize Deliverables

## Expected Outputs
- design chain roadmap
- project schedule

## Business Rules
- Design Chain Roadmap must be reviewed and updated regularly
- Resource Utilization must be tracked and reported
- Milestone Achievement must be measured against Design Chain Objectives

## Exception Handling
- IF Design Chain Strategy is not available THEN use industry benchmarks to create Design Chain Roadmap
- IF Resource Availability is unknown THEN assume default resource allocation
- IF Milestone Achievement is not measurable THEN use proxy metrics to evaluate progress

## Success Criteria
- Design Chain Roadmap is completed and approved
- Milestones are achieved on time and within budget
- Resource Utilization is optimized

## Compliance Requirements
- GxP if pharma or medical devices
