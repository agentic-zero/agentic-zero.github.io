# SOP — Manage Design Data and Intellectual Property
**Process ID:** SCOR-D1.5
**Framework:** SCOR-D | **Domain:** Design Chain
**Generated:** 2026-06-05

## Purpose
Process of managing design data and intellectual property across the organization, including data storage, security, and access control

## Triggers
- New Design Data is created
- Intellectual Property is acquired or developed
- Changes to Organization's structure or policies occur
- Security threats or breaches are detected

## Inputs Required
- design data
- intellectual property

## Process Steps
1. IF Design Data is sensitive THEN apply additional Security measures
2. IF Intellectual Property is high-risk THEN implement strict Access Control
3. IF data breach is detected THEN trigger incident response protocol

## Expected Outputs
- managed design data
- protected intellectual property

## Business Rules
- All Design Data must be stored in secure Data Storage
- Access to Design Data must be restricted to authorized personnel
- Intellectual Property must be protected from unauthorized use or disclosure
- Compliance with GxP, EU AI Act, and GDPR regulations must be ensured when applicable

## Exception Handling
- IF data loss occurs THEN restore from backup and notify stakeholders
- IF unauthorized access is detected THEN trigger incident response protocol and review Access Control
- IF Intellectual Property is compromised THEN initiate damage control and review protection measures

## Success Criteria
- Design Data is accurately and securely stored
- Intellectual Property is protected from unauthorized use or disclosure
- Access Control is effective in regulating access to Design Data
- Compliance with relevant regulations is ensured

## Compliance Requirements
- GxP if pharma or medical devices
- EU AI Act if AI-driven design
- GDPR if personal data
