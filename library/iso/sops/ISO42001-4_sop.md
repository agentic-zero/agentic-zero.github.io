# SOP — AI Context — Organization and Interested Parties
**Process ID:** ISO42001-4
**Framework:** ISO 42001:2023 | **Domain:** ISO 42001
**Generated:** 2026-06-10

## Purpose
Understanding the organizational context for AI management including AI-specific interested parties, regulatory environment, AI use case scope and AIMS (AI Management System) boundary definition

## Triggers
- New AI use case inventory received
- Regulatory requirements update event
- Organizational AI strategy revision

## Inputs Required
- AI use case inventory
- regulatory requirements
- stakeholder AI expectations
- organizational AI strategy

## Process Steps
1. IF all 4 inputs present THEN execute context analysis
2. IF regulatory requirements updated THEN refresh AIContextAnalysis
3. IF sector in sector_applicability list THEN include in AIMSScope

## Expected Outputs
- AIMS scope
- AI interested party register
- AI context analysis
- AI use case register

## Business Rules
- Every output must reference at least one compliance_flag
- KPI:RegulatoryComplianceCoverage must be calculated from regulatory requirements input
- Process must record source and confidence values on every output

## Exception Handling
- If automation_potential < 0.6 require manual stakeholder sign-off before producing outputs
- If any input missing then halt and log exception before generating AIMSScope

## Success Criteria
- All 4 outputs generated with non-null values
- KPI:AIUseCaseCoverage >= 0.9
- All compliance_flags attached to AIContextAnalysis

## Compliance Requirements
- ISO 42001:2023 Clause 4
- EU AI Act context assessment
- GDPR AI context
