# SOP — Manage Supplier Information and Performance
**Process ID:** SCOR-S1.3
**Framework:** SCOR | **Domain:** Source
**Generated:** 2026-06-02

## Purpose
Process of managing supplier information and evaluating supplier performance

## Triggers
- new Supplier registration
- regular review of Supplier performance (e.g. quarterly)

## Inputs Required
- supplier information
- performance data
- quality metrics

## Process Steps
1. IF Supplier quality rating is below threshold THEN initiate Supplier Development Plan
2. IF Supplier delivery performance is below threshold THEN reassess Supplier Scorecard

## Expected Outputs
- supplier scorecards
- supplier development plans

## Business Rules
- rule1: Supplier Information must be up-to-date and accurate
- rule2: Performance Data and Quality Metrics must be collected regularly
- rule3: Supplier Scorecards must be reviewed and updated quarterly

## Exception Handling
- exception1: Missing or incomplete Supplier Information - handle by requesting additional information from Supplier
- exception2: Discrepancy in Performance Data or Quality Metrics - handle by investigating and resolving discrepancy

## Success Criteria
- Supplier quality rating meets or exceeds threshold
- Supplier delivery performance meets or exceeds threshold
- Supplier Development Plans are implemented and effective

## Compliance Requirements
- GxP if pharma
- GDP if distribution
